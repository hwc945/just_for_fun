# 项目规划与演进路线图

本文档旨在记录当前项目的不足之处，并规划未来的优化方向和功能扩展。

## 🛑 当前缺点与痛点 (Current Drawbacks)

1.  **数据源不稳定**:
    *   目前依赖免费的第三方 API (`api.yujn.cn`)，随时可能失效、限流或返回重复数据。
    *   缺乏数据校验机制，视频格式或编码不兼容时会导致播放失败（虽然已添加跳过逻辑，但体验仍受损）。
2.  **缺乏本地缓存**:
    *   每次播放都需要重新下载视频，消耗大量用户流量。
    *   弱网环境下体验极差，视频卡顿明显。
3.  **状态管理简陋**:
    *   目前完全依赖 `setState` 和 `StatefulWidget` 进行状态管理。
    *   随着功能增加（如点赞、评论、用户信息），代码耦合度会急剧上升，难以维护。
4.  **音频管理缺失**:
    *   没有处理音频焦点（Audio Focus）。如果在播放音乐时打开应用，声音会混杂在一起。
5.  **UI/UX 细节不足**:
    *   视频加载前显示黑屏或简单的转圈，缺乏平滑的过渡（如封面图占位）。
    *   没有沉浸式状态栏，顶部状态栏遮挡了部分视频内容。

## ⚡ 短期优化目标 (Short-term Optimizations)

这些功能可以在短期内（1-2周）实现，能显著提升用户体验。

1.  **沉浸式体验**:
    *   使用 `SystemChrome.setSystemUIOverlayStyle` 设置状态栏透明。
    *   调整 UI 布局，避开刘海屏和底部 Home 条区域 (`SafeArea`)。
2.  **双击点赞**:
    *   实现双击屏幕出现爱心动画的效果。
    *   使用 `AnimationController` 和 `Overlay` 或 `Stack` 实现。
3.  **视频封面占位**:
    *   在视频加载完成前，先显示视频的缩略图（如果 API 提供）或高斯模糊背景，避免黑屏。
4.  **音量与亮度手势**:
    *   在屏幕左侧上下滑动调节亮度，右侧上下滑动调节音量。
5.  **网络状态检测**:
    *   使用 `connectivity_plus` 库检测网络状态。
    *   非 WiFi 环境下提示用户流量消耗。

## 🔭 长期演进方向 (Long-term Vision)

这些是架构层面的改进，适合在项目中期进行。

1.  **引入状态管理框架**:
    *   迁移至 **Provider** 或 **Bloc**。
    *   将业务逻辑（视频获取、播放控制）与 UI 分离。
2.  **本地缓存系统**:
    *   引入 `flutter_cache_manager`。
    *   实现视频文件的本地缓存，再次播放时直接读取本地文件。
    *   实现预下载策略：在 WiFi 环境下自动下载后续 5-10 个视频。
3.  **音频焦点管理**:
    *   引入 `audio_session`。
    *   实现与其他音频应用（如音乐播放器）的互斥播放逻辑。
4.  **国际化 (i18n)**:
    *   支持多语言切换，为发布到不同地区做准备。

## 🚀 未来新增大模块 (Future Modules)

这些是扩展应用核心价值的重大功能模块。

1.  **用户系统 (User System)**:
    *   注册/登录（手机号、第三方登录）。
    *   个人主页：展示头像、昵称、简介。
    *   作品列表：展示用户发布或喜欢的视频。
2.  **社交互动 (Social Interaction)**:
    *   **点赞系统**: 记录用户喜欢的视频，并同步到服务端。
    *   **评论系统**: 底部弹出评论面板，支持多级评论。
    *   **分享功能**: 生成分享链接或海报，分享到微信/QQ/系统分享。
3.  **内容创作 (Content Creation)**:
    *   **视频拍摄**: 调用摄像头录制视频。
    *   **视频编辑**: 简单的剪辑、滤镜、添加背景音乐。
    *   **视频上传**: 将创作的内容上传到云存储（如 OSS/S3）。
4.  **后端服务集成 (Backend Integration)**:
    *   开发或对接真正的后端服务（Go/Node.js/Java）。
    *   实现推荐算法：根据用户行为（观看时长、点赞）推荐感兴趣的视频，而不是随机获取。
